# 自检开始文档：第2点能力可靠性与投稿级可用性加固

时间：2026-02-27
负责人：Codex

## 目标
针对已实现的 CultureMERT / 风格迁移 / 动态本体进行自检，修复影响“真实可用”和“论文可信度”的关键问题。

## 自检发现（优先级）
1. 风格迁移诊断指标存在缺陷：`zc_drift/za_drift` 实际恒为 0（比较对象错误）。
2. 数据构建缺少可追溯清单：无 manifest、无重复 track_id 显式校验。
3. 动态本体尚未直接服务训练：缺少“本体标注 -> pairwise constraints”导出能力。

## 本轮加固范围
- 修复风格迁移诊断逻辑并补充风格转移度指标
- 为音频建库增加 manifest 与数据一致性校验
- 新增本体约束导出（CLI + API）以直接回灌训练

## 验收标准
- 风格迁移 meta 指标不再出现结构性恒零
- 数据构建产出 manifest，且重复 track_id 能被拦截
- 本体可导出 jsonl 约束文件，供训练直接使用
