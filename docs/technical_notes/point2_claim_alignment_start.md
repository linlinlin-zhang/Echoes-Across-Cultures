# 第2点开始文档：论文主张与实现一致性修复

时间：2026-02-27
负责人：Codex

## 目标
解决“文稿主张范围大于当前实现能力”导致的学术可信度风险，使仓库对外叙述与代码证据一一对应。

## 问题描述
当前仓库已经是可运行原型，但在研究叙事中容易给出超出实现范围的印象，例如：
- CultureMERT 真实接入与实验尚未在仓库形成完整流程
- 解纠缠评估（如 MIG/DCI）与公平性评估未形成统一评测脚本
- 生成式风格迁移为规划项，尚未提供实装模块

这会在论文审稿中触发“claim-evidence mismatch”（主张-证据不一致）。

## 本次修复策略
1. 在 README 中新增“已实现/部分实现/计划中”能力矩阵
2. 新增 `docs/PAPER_CLAIM_ALIGNMENT.md`，逐条映射研究主张与代码证据
3. 明确投稿时可主张与不可主张内容，降低过度陈述风险

## 验收标准
- 读者仅凭仓库文档即可明确当前能力边界
- 每项核心主张均有对应文件路径证据或明确标注为“计划中”
- 不再出现“实现状态不明”区域
